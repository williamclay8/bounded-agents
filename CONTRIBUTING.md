# Contributing

This project is meant to stay small and practical.

Good contributions:

- clearer adoption examples;
- safer cost-gate patterns;
- better receipt templates;
- canaries for common agent runners;
- examples that separate local, committed, pushed, and deployed/live state.

Please avoid:

- claims that the workflow guarantees correctness;
- live provider or billing integrations without a scrubbed read-only path;
- public posting, deploy, wallet, payment, or email automation examples that skip approval;
- framework-specific rewrites that hide the core pattern.
